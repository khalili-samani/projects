# Australian Energy Market Analysis Pipeline

An end-to-end ETL and analytics pipeline that collects, processes, stores and analyses wholesale electricity market data from the Australian National Electricity Market (NEM).

The project transforms raw operational market data into structured analytical datasets, SQL reporting views and automated visual reports, demonstrating practical data engineering, SQL and analytics skills using real-world Australian energy market data.

![Daily Average Wholesale Price](outputs/charts/daily_avg_rrp_by_region_202603.png)

---

## Business Problem

Electricity markets generate large volumes of operational data every day. Market analysts, business analysts and energy professionals rely on accurate reporting to understand pricing behaviour, demand patterns and regional market performance.

Raw data alone provides limited value. Before meaningful analysis can occur, data must be collected, cleaned, validated, structured and transformed into reporting-ready datasets.

This project automates that process by building a repeatable analytics pipeline that converts raw Australian Energy Market Operator (AEMO) data into business-focused insights and visual reports.

## Key Insights

The March 2026 dataset revealed several notable market characteristics:

### NSW1

* Generally exhibited the highest average prices
* Demonstrated relatively stable demand behaviour
* Experienced fewer negative pricing events

### QLD1

* Showed moderate volatility
* Demonstrated several high-price periods
* Demand remained relatively consistent throughout the month

### VIC1

* Recorded the highest price volatility
* Experienced the largest proportion of negative pricing intervals
* Displayed the widest distribution of wholesale electricity prices

These observations highlight how regional market dynamics can differ significantly despite operating within the same national market.

## Solution Overview

The project was designed as a complete analytics workflow rather than a single script.

The pipeline consists of five stages:

| Stage               | Purpose                                |
| ------------------- | -------------------------------------- |
| Data Acquisition    | Download AEMO market files             |
| Data Cleaning       | Validate and standardise datasets      |
| Database Loading    | Store structured data in MySQL         |
| SQL Reporting Layer | Create reusable analytical views       |
| Visual Reporting    | Generate automated charts and insights |

Together, these components demonstrate the full lifecycle of a reporting solution.

---

## Database Design

The database uses a central fact table designed for analytical workloads.

### Fact Table

`fact_nem_price_demand`

| Column              | Description                   |
| ------------------- | ----------------------------- |
| region_code         | NEM region identifier         |
| settlement_datetime | 5-minute settlement timestamp |
| trading_date        | Trading date                  |
| rrp_aud_mwh         | Regional Reference Price      |
| total_demand_mw     | Electricity demand            |
| period_type         | Settlement period type        |

Indexes support efficient reporting by region and date.

## Running the Project

### Install Dependencies

```bash
pip install -r requirements.txt
```

### Configure Environment Variables

```powershell
$env:MYSQL_USER="root"
$env:MYSQL_PASSWORD="your_password"
$env:MYSQL_HOST="localhost"
$env:MYSQL_DATABASE="aus_energy_automation"
```

### Run the Pipeline

```bash
python scripts/run_pipeline.py
```

The pipeline automatically:

1. Downloads market data
2. Cleans and validates records
3. Loads data into MySQL
4. Creates reporting views
5. Generates analytical charts
